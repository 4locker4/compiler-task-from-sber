## RISC Instruction Encoder

Автоматическое построение системы кодирования команд для RISC-подобной архитектуры на основе списка требований в формате JSON.
## Описание

Скрипт читает JSON-файл с описанием форматов инструкций, полей и списка команд, автоматически распределяет битовые поля согласно заданным ограничениям и генерирует кодировку для каждой инструкции.

## Установка и запуск

```
git clone <repository-url>
cd <project-directory>
ruby main.rb <input.json>
```

Результат сохраняется в файл result.json в текущей директории.
Формат входного JSON

## Обязательные поля входного JSON

|Поле	| Тип	| Описание|
|-------|-------|---------|
|length  | число	| Длина инструкции в битах|
|fields	| массив	| Список полей с их размерами|
|instructions	|массив	| Описание форматов инструкций|

Поля в секции fields

Каждый элемент массива — объект с одним ключом: именем поля и его размером.
text

{ "R0" : "5" },
{ "imm" : ">=8" }

Поля в секции instructions
Поле	Тип	Описание
insns	массив	Список инструкций данного формата
operands	массив	Список операндов (имена полей)
format	строка	Название формата
comment	строка	Комментарий (опционально)
Пример входного файла
text

{
  "length" : "25",
  "fields" : [
    { "R0" : "5" },
    { "R1" : "5" },
    { "R2" : "5" },
    { "imm" : ">=8" },
    { "disp" : ">=12" },
    { "code" : ">=3" }
  ],
  "instructions" : [
    {
      "insns" : ["add", "sub", "mul", "div", "and", "or", "xor", "lsl", "lsr", "asr", "cmp", "call"],
      "operands" : ["R0", "R1", "R2"],
      "format" : "alu_instructions",
      "comment" : "3-operand arithmetic and logic instructions, function call"
    },
    {
      "insns" : ["load.b", "store.b", "load.d", "store.d", "load.w", "store.w", "branch", "jump", "zip", "move"],
      "operands" : ["R0", "R1", "imm"],
      "format" : "memory_access",
      "comment" : "Instructions for reading and writing memory, short branches"
    },
    {
      "insns" : ["branch.cond"],
      "operands" : ["R0", "code", "disp"],
      "format" : "branch",
      "comment" : "branch instruction"
    }
  ]
}

Формат выходного JSON

Выходной файл содержит массив объектов, каждый из которых описывает одну инструкцию:
text

{
  "insn" : "add",
  "format" : "alu_instructions",
  "fields" : [
    { "F" : {"msb" : 24, "lsb" : 23, "value" : "00"} },
    { "OPCODE" : {"msb" : 22, "lsb" : 19, "value" : "0000"} },
    { "R0" : {"msb" : 18, "lsb" : 14, "value" : "+"} },
    { "R1" : {"msb" : 13, "lsb" : 9, "value" : "+"} },
    { "R2" : {"msb" : 8, "lsb" : 4, "value" : "+"} },
    { "RES0" : {"msb" : 3, "lsb" : 0, "value" : "0000"} }
  ]
}

Поля выходного формата

    insn — название инструкции

    format — название формата (из входного файла)

    fields — массив полей с указанием расположения битов:

        msb — старший бит поля

        lsb — младший бит поля

        value — значение фиксированных битов; для переменных операндов указывается "+"

Алгоритм работы

    Парсинг входного JSON — чтение, без валидации (по ТЗ)

    Анализ форматов — определение количества бит для:

        Поля формата (F) — идентификация типа инструкции

        Поля кода операции (opcode) — различение инструкций внутри формата

        Операндов — согласно заданным размерам

    Распределение битов — последовательное размещение полей от старших битов к младшим

    Расширение полей — при наличии свободных битов они распределяются в поля с минимальным размером (">=")

    Заполнение остатка — неиспользованные биты помещаются в поля RES0, RES1 и т.д.

    Генерация кодировок — для каждой инструкции формируются фиксированные значения полей F и opcode
